## How to add a new entity type ‚Äî always follow this pattern

```typescript
// entities/patient.ts
import { z } from 'zod'

// 1. Zod schema first ‚Äî single source of truth
export const PatientSchema = z.object({
  id: z.string().uuid(),
  name: z.string().min(1, 'Nome √© obrigat√≥rio'),
  cpf: z.string().length(11, 'CPF deve ter 11 d√≠gitos'),
  phone: z.string().nullable(),
  email: z.string().email().nullable(),
  birthDate: z.string().nullable(),         // ISO date string
  convenioId: z.string().uuid().nullable(),
  active: z.boolean().default(true),
  deletedAt: z.string().nullable(),         // soft delete
  createdAt: z.string(),
  updatedAt: z.string(),
})

// 2. TypeScript type derived from schema ‚Äî never write manually
export type Patient = z.infer<typeof PatientSchema>

// 3. Create/update schemas omit server-managed fields
export const CreatePatientSchema = PatientSchema.omit({
  id: true,
  deletedAt: true,
  createdAt: true,
  updatedAt: true,
})

export const UpdatePatientSchema = CreatePatientSchema.partial()

export type CreatePatientInput = z.infer<typeof CreatePatientSchema>
export type UpdatePatientInput = z.infer<typeof UpdatePatientSchema>
```

## Naming conventions ‚Äî always enforce

- Types: `PascalCase` ‚Äî `Patient`, `Appointment`, `AttendanceProcedure`
- Schemas: `PascalCase` + `Schema` suffix ‚Äî `PatientSchema`, `CreatePatientSchema`
- Input types: `PascalCase` + `Input` suffix ‚Äî `CreatePatientInput`, `UpdatePatientInput`
- Enums: `PascalCase` ‚Äî `UserRole`, `AppointmentStatus`
- Enum values: `SCREAMING_SNAKE_CASE` ‚Äî `UserRole.ADMIN`, `AppointmentStatus.CANCELLED`

## Supabase generated types

`src/supabase.ts` is auto-generated. Regenerate after every migration:

```bash
supabase gen types typescript --local > packages/types/src/supabase.ts
```

The `Database` type from this file is the ground truth for table shapes. Domain entity types in `entities/` should align with it but can be more ergonomic (camelCase, nullable handling, etc.).

## Exporting ‚Äî always update index.ts

Every new type/schema added must be re-exported from `src/index.ts`:

```typescript
export * from './entities/patient'
export * from './entities/appointment'
// etc.
```
